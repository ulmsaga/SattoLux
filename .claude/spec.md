# SattoLux — 프로젝트 Spec

## 개요

주말 번호 생성기 (가족 공유용 단일 계정 기준)

## 기술 스택

### Front-End
- React + **Vite 4.x** (Node 16 한계로 Vite 5 사용 불가)
- Tailwind CSS
- shadcn/ui
- AG-Grid
- Axios **1.15.0 이상** (보안 이슈로 버전 고정)

### Back-End
- Spring Boot + Java 17
- MyBatis (`sqlSessionTemplate` + namespace 방식, **JPA 사용 안 함**)
- MySQL (Docker)

### LLM 연동
- **Anthropic Claude API** — `CLAUDE_API_KEY` 사용
- HOT_NUMBER / MIXED 규칙의 번호 조합 추천 시 활용

## 공통 요구사항

### 반응형
- Phone (메인 기준)
- Desktop PC

### 레이아웃 구조
```
[top bar]  ← 좌측 햄버거(삼발이) 버튼
[menu] | [메인 화면]
```
- 햄버거 버튼 클릭 → 좌측 상단에서 메뉴 슬라이드 in/out
- 메뉴는 1LV 단층 구조 (심플)
- Default: 메뉴 비활성

### 화면 영역 구분
- **LoginLayer**: 로그인 전용
- **MainLayer**: 로그인 인증 후 진입

## 기능 명세

### 화면 1 — 번호 생성 및 조회
- 매주 지정 요일(기본: 목요일)에 N세트 자동 생성
- 자동 생성은 **설정된 요일 + properties에 정의된 스케줄 시각** 기준으로 실행
- 스케줄 실행 시각의 초기값은 **오전 9시**로 하며, properties로 관리
- 주간 총 세트 수량: 1 / 5 / 10 / 15 / 20 (기본: 10세트)
- 해당 요일에 앱 진입 시 그주 번호가 이미 세팅되어 있음 (알림 없이 조회 방식)
- 생성 방식: 설정 row 조합으로 구성 (예: 완전랜덤 5세트 + HOT_NUMBER 5세트)
- 설정된 요일에 properties 기준 시각이 지났는데도 이번 주 번호가 생성되지 않은 경우, 화면에서 **수동 생성 버튼을 활성화**해 사용자가 직접 생성할 수 있어야 함

#### 번호 생성 설정값 (`generation_rule`)
| 항목 | 기본값 | 설명 |
|------|--------|------|
| 생성 요일 | 목요일 | 월~금 중 선택, 자동 생성 트리거 요일 |
| 생성 전략 | RANDOM | `RANDOM`, `HOT_NUMBER`, `MIXED` |
| 생성 엔진 | LOCAL | `LOCAL`, `CLAUDE` |
| 세트 수 | 10 | 해당 row가 생성할 세트 수 |
| 분석 회차 수 | 1000 | `HOT_NUMBER`, `MIXED` 전략에서 사용할 과거 회차 수 |
| 사용 여부 | Y | row 활성/비활성 |

- 사용자별로 설정 row를 여러 건 가질 수 있음
- 같은 요일에 여러 row를 둘 수 있음
- 한 주 총 세트 수는 row 합계 기준으로 1 / 5 / 10 / 15 / 20 검증
- `RANDOM`은 완전 랜덤 전략
- `HOT_NUMBER`는 최근 N회에서 자주 나온 번호를 pool로 만든 뒤 조합 생성
- `MIXED`는 RANDOM + HOT_NUMBER 성격을 함께 반영하는 혼합 전략
- `LOCAL`은 서버 내부 로직으로 생성, `CLAUDE`는 Claude API 응답으로 생성
- 자동 생성 스케줄 시각은 사용자별 설정값이 아니라 **서버 전역 properties 값**으로 관리

### 화면 2 — 추첨 결과 비교
- 토요일 추첨 결과를 서치해서 기 생성 세트와 비교
- 결과를 화면에 노출
- 추가 검토: **무료 Noti 방법 파악 필요** (카카오 등)

### 화면 1-1 — 마킹 보기
- 번호 생성 완료 후 `마킹 보기` 버튼으로 진입
- 현재 주차 생성 번호를 실제 로또 마킹지 형태로 보여줌
- `5세트 = 1장` 기준으로 해석
- `한 화면 = 한 구역(A~E) = 한 세트`
- 좌우 드래그로 세트 단위 연속 이동
- 이동 규칙
  - `1장 A → 1장 B → 1장 C → 1장 D → 1장 E`
  - `1장 E` 다음은 `2장 A`
  - `2장 A`에서 이전은 `1장 E`
- 상단에 현재 `장 번호`, `구역 문자`, `전체 세트 순번`을 함께 표시
- 실제 사용 목적은 복권 용지에 손으로 마킹할 때 참고하기 위함

## 인증/인가 흐름

### 사전 단계
1. RSA 공개키 발급

### 1차 인증 — 비밀번호 검증
- RSA 암호화된 비밀번호 전송
- 서버는 계정 상태, 잠금 여부, 실패 횟수를 함께 검증

### 2차 인증 — OTP (선택적 적용)
- **Skip 가능**: 사용자/환경에 따라 2차 인증 비활성화 옵션
- 카카오 알림톡 무료 플랜 가능 시 사용
- 불가 시 → **mailpit** (메일 기반)
- OTP는 원문 저장하지 않고 해시 저장

### JWT 발급 및 세션 유지
- JWT 발급 후 SSE 연결
- Refresh Token 로테이션으로 세션 유지
- Refresh Token은 원문 저장하지 않고 해시 저장

### 로그아웃
- 클라이언트 토큰 제거 + 서버 무효화

### 추후 구현 (현재 제외)
- 중복 로그인 처리
- 중복 로그인 강제 로그아웃 흐름
