# SattoLux — 프로젝트 Spec

## 개요

주말 로또 번호 생성기 (사용자: 3명, 확장 가능)

## 기술 스택

### Front-End
- React + **Vite 4.x** (Node 16 한계로 Vite 5 사용 불가)
- Tailwind CSS
- shadcn/ui
- AG-Grid
- Axios **1.15.0 이상** (보안 이슈로 버전 고정)

### Back-End
- Spring Boot + Java 17
- MyBatis (`sqlSessionTemplate` + namespace 방식, **JPA 사용 안 함**)
- MySQL (Docker)

### AI 연동
- **OpenAI API (GPT-4o)** — Codex CLI용 `OPENAI_API_KEY` 재사용
- 번호 생성 시 분석 기반 세트에 활용 (사용자 설정으로 비율 조정 가능)

## 공통 요구사항

### 반응형
- Phone (메인 기준)
- Desktop PC

### 레이아웃 구조
```
[top bar]  ← 좌측 햄버거(삼발이) 버튼
[menu] | [메인 화면]
```
- 햄버거 버튼 클릭 → 좌측 상단에서 메뉴 슬라이드 in/out
- 메뉴는 1LV 단층 구조 (심플)
- Default: 메뉴 비활성

### 화면 영역 구분
- **LoginLayer**: 로그인 전용
- **MainLayer**: 로그인 인증 후 진입

## 기능 명세

### 화면 1 — 번호 생성 및 조회
- 매주 지정 요일(기본: 목요일)에 N세트 자동 생성
- 세트 수량: 5 / 10 / 15 / 20 (기본: 10세트)
- 해당 요일에 앱 진입 시 그주 번호가 이미 세팅되어 있음 (알림 없이 조회 방식)
- 생성 방식: 사용자 설정값으로 구성 (예: 완전랜덤 5세트 + AI분석 5세트)

#### 번호 생성 설정값 (user_config)
| 항목 | 기본값 | 설명 |
|------|--------|------|
| 생성 요일 | 목요일 | 자동 생성 트리거 요일 |
| 총 세트 수 | 10 | 5 / 10 / 15 / 20 선택 |
| 랜덤 세트 수 | 5 | 완전 랜덤 생성 |
| AI 분석 세트 수 | 5 | 최근 N회 분석 기반 생성 |
| AI 분석 회차 수 | 1000 | 분석에 사용할 과거 회차 수 |
| AI 제공자 | OpenAI GPT-4o | Codex CLI `OPENAI_API_KEY` 재사용 |

### 화면 2 — 추첨 결과 비교
- 토요일 추첨 결과를 서치해서 기 생성 세트와 비교
- 결과를 화면에 노출
- 추가 검토: **무료 Noti 방법 파악 필요** (카카오 등)

## 인증/인가 흐름

### 사전 단계
1. RSA 공개키 발급

### 1차 인증 — 비밀번호 검증
- RSA 암호화된 비밀번호 전송

### 2차 인증 — OTP (선택적 적용)
- **Skip 가능**: 사용자/환경에 따라 2차 인증 비활성화 옵션
- 카카오 알림톡 무료 플랜 가능 시 사용
- 불가 시 → **mailpit** (메일 기반)

### JWT 발급 및 세션 유지
- JWT 발급 후 SSE 연결
- Refresh Token 로테이션으로 세션 유지

### 로그아웃
- 클라이언트 토큰 제거 + 서버 무효화

### 추후 구현 (현재 제외)
- 중복 로그인 처리
- 중복 로그인 강제 로그아웃 흐름
